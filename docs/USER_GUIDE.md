# 网银账单小助手 用户使用说明

版本：`1.2.10`
更新日期：`2026-03-11`

## 一、软件用途

本软件用于将网银导出的原始账单文件，按既定模板转换为可直接使用的明细账单和余额账单。

当前包含 2 个模块：

1. `网银账单生成`
2. `新开账户生成网银账单`

## 二、启动后界面说明

1. 左上角有模块切换按钮 `🔁`，可在两个模块之间切换。
2. 右下角会显示当前版本号。
3. 左下角有 `🎨` 调色盘按钮，可修改背景颜色或导入背景图片。

## 三、模块一：网银账单生成

### 适用场景

用于处理已有交易明细的网银账单文件，生成：

1. 明细账单
2. 余额账单

### 使用步骤

#### 第 1 步：导入模板

1. 点击 `导入模板`。
2. 选择模板文件，支持 `Excel / CSV`。
3. 导入成功后，模板会出现在主界面的 `模板` 下拉框中。

#### 第 2 步：维护映射关系

1. 点击主界面的 `模板管理`。
2. 在模板列表中找到对应模板，点击行内的 `修改`。
3. 在弹出的 `映射关系管理` 中维护字段关系。

### 映射规则说明

#### 1. 基础规则

1. 允许多个模板字段映射到同一个原始字段。
2. 映射字段可以留空；留空时，该字段仍会保留在导出的明细里，但值为空。
3. `Balance` 默认值为 `无`。
4. `Balance` 为 `无` 时，不生成余额账单。

#### 2. Channel

1. `Channel` 不需要手动映射。
2. 软件会自动取 `模板名称中 - 前面的内容` 作为 `Channel`。
3. 例如模板名为 `中国银行-香港`，则 `Channel` 固定为 `中国银行`。

#### 3. MerchantId

1. 可以直接映射导入文件中的某个字段。
2. 也可以选择 `自己输入`，然后填写固定文本。
3. 一旦选择 `自己输入`，该模板所有涉及 `MerchantId` 的地方都使用这个固定值。
4. 如果该模板启用了 `Balance`，则 `MerchantId` 为余额链路必填项：
   1. 必须已映射。
   2. 导入文件中每条参与余额生成的记录，`MerchantId` 最终值不能为空。

#### 4. Currency

1. 可以直接映射导入文件中的某个字段。
2. 也可以选择 `自己输入`，然后填写固定文本。
3. 一旦选择 `自己输入`，该模板所有涉及 `Currency` 的地方都使用这个固定值。

#### 5. 根据发生额做映射

在 `映射关系管理` 底部，新增两条辅助规则：

1. `根据发生额做映射的户名`
2. `根据发生额做映射的账户号`

作用说明：

1. 当一行存在 `Credit Amount` 时：
   1. `Drawee Name` 取“根据发生额做映射的户名”对应字段的值。
   2. `Drawee CardNo` 取“根据发生额做映射的账户号”对应字段的值。
2. 当一行存在 `Debit Amount` 时：
   1. `Payee Name` 取“根据发生额做映射的户名”对应字段的值。
   2. `Payee Cardno` 取“根据发生额做映射的账户号”对应字段的值。

注意：

1. 这两条是辅助映射规则。
2. 它们不会作为列导出到明细文件里。

#### 第 3 步：维护账户映射（可选）

适用场景：

如果你希望把原始网银大账户 ID 替换成清结算系统大账户 ID，可以使用 `账户映射`。

操作方式：

1. 点击 `账户映射`。
2. 左侧填写 `网银大账户ID`。
3. 右侧填写 `清结算系统大账户ID`。
4. 保存后，凡是映射为 `MerchantId` 且未选择 `自己输入` 的字段，都会按此表替换。

#### 第 4 步：选择模板

1. 在主界面的 `模板` 下拉框中选择你要使用的模板。

#### 第 5 步：导入原始账单文件

1. 点击 `导入文件`。
2. 选择原始账单文件，支持 `Excel / CSV`。
3. 软件会先生成明细账单，并在满足条件时继续生成余额账单。

### 导入后的处理规则

#### 1. 明细字段输出

1. 明细字段会按模板映射输出。
2. 所有模板字段都会保留在导出的明细中，即使没有映射或原值为空，也会保留该列。
3. 导出的明细账单不会保留 `Balance` 列。

#### 2. Credit Amount / Debit Amount

1. 原始值会先自动清洗，只保留数字 `0-9` 和小数点 `.`。
2. 输出为数值格式。
3. 例如：
   1. `$1,234.56CR -> 1234.56`
   2. `DB789.01元 -> 789.01`

校验规则：

1. 如果一条记录的 `Credit Amount` 与 `Debit Amount` 同时为 `0` 或空值，该记录不会参与导出的明细账单和余额账单生成。
2. 如果一条记录的 `Credit Amount` 与 `Debit Amount` 同时都有值，系统会中止本次导入，并生成详细报错文件。

#### 3. Currency

1. 如果原值本身是英文，则直接输出。
2. 如果原值不是英文，软件会自动使用内置 `币种映射表.xlsx` 做模糊匹配。
3. 匹配到后，自动替换为英文简称，例如：`美元 -> USD`。
4. 如果匹配不到：
   1. 软件仍会保留原值并继续生成文件。
   2. 同时会生成详细报错文件，状态框可点击导出。

#### 4. Balance

1. 如果映射到源字段，原始值会自动清洗，只保留数字 `0-9` 和小数点 `.`。
2. 也可以在映射关系管理中将 `Balance` 设为 `通过发生额计算`。
3. 设为 `通过发生额计算` 后，系统会按 `上一账单日余额 + 当日 Credit Amount 汇总 - 当日 Debit Amount 汇总` 生成余额。

### 导出文件

#### 1. 导出明细

点击 `导出明细`，导出本次生成的明细账单。

命名规则：

`模板名-COMMON-最早账单日期~最晚账单日期.xlsx`

如果只有一个账单日期，则不带 `~`。

#### 2. 导出余额

点击 `导出余额`，导出本次生成的余额账单。

命名规则：

`模板名-Balance-最早账单日期~最晚账单日期.xlsx`

如果只有一个账单日期，则不带 `~`。

### 余额账单生成说明

#### 1. 生成条件

只有当 `Balance` 已映射且不为 `无` 时，才会尝试生成余额账单，包括：

1. 映射到源文件字段
2. 设为 `通过发生额计算`

#### 2. 余额账单字段来源

1. `银行名称` = 模板名称中 `-` 前面的内容
2. `所在地` = 模板名称中 `-` 后面的内容
3. `币种` = 明细中的 `Currency`
4. `银行账号` = 明细中的 `MerchantId`
5. `账单日期` = 明细中的 `BillDate`
6. `期末余额`：
   1. 如果 `Balance` 映射到源字段，则取明细中的 `Balance`
   2. 如果 `Balance` 设为 `通过发生额计算`，则按 `上一账单日余额 + 当日 Credit Amount 汇总 - 当日 Debit Amount 汇总` 生成
7. `期初余额`、`期初可用余额`、`期末可用余额` 默认留空

#### 3. 同一账单日有多个余额时的处理

如果同一账单日存在多个 `Balance`：

1. 软件会优先尝试根据上一账单日余额和当日收支推导期末余额。
2. 如果当前文件里没有可用的上一账单日余额：
   1. 软件会先读取本地余额种子文件。
   2. 若本地余额种子也取不到，则不会立即报错。
   3. 状态框会提示：`因首次导入余额，请导入上一个账单日余额用于余额校验`

#### 4. 本地余额种子

本地余额种子只用于余额校验，不参与明细导出。

存储位置：

`文档/网银账单生成小助手/balance-seeds/`

存储规则：

1. 按银行拆分文件。
2. 每条记录按以下键存储：
   1. `MerchantId`
   2. `Currency`
   3. `BillDate`
3. 每条记录还会带 `生成方式`，用于区分 `账单里的余额`、`通过发生额计算`、`人工录入`。
4. 如果你再次录入相同日期、相同账号、相同币种的余额，系统会先提示确认是否覆盖。

#### 5. 状态框补录上一账单日余额

当状态框提示：

`因首次导入余额，请导入上一个账单日余额用于余额校验`

说明：

1. 明细账单已经生成，仍可导出。
2. 余额账单暂时还未生成。
3. 此时状态框点击行为不是导出报错文件，而是打开余额补录窗口。

补录步骤：

1. 点击状态框。
2. 在弹窗中：
   1. 选择 `上一账单日日期`
   2. 输入 `上一账单日余额`
3. 点击 `完成`。
4. 软件会先把这条余额写入本地余额种子文件。
5. 然后立即重试当前这次余额校验。
6. 如果校验成功，状态会变成 `明细账单可导出，余额账单可导出`。

#### 6. 什么情况下仍会报错

以下情况仍属于真实报错：

1. `MerchantId` 缺少映射
2. 启用了 `Balance`，但导入记录里的 `MerchantId` 为空
3. 补录的余额不是合法数字
4. 补录后仍然无法匹配到正确的期末余额
5. 其他字段或文件本身存在格式问题

## 四、模块二：新开账户生成网银账单

### 适用场景

仅支持 `开户至今未发生交易` 的账户余额导出。

### 使用步骤

1. 点击左上角 `🔁`，切换到 `新开账户生成网银账单`。
2. 依次填写以下必填项：
   1. 银行名称
   2. 所在地
   3. 币种
   4. 银行账号
   5. 开户日期
3. 所有必填项填写完成后，`生成` 按钮才可点击。
4. 点击 `生成` 后，软件会按内置余额账单模板生成新的余额账单文件。
5. 点击 `导出` 可将该文件另存为到指定位置。

### 币种填写规则

#### 单币种账户

1. 直接在币种输入框中填写币种文本。

#### 多币种账户

1. 勾选 `多币种账户`。
2. 币种输入框会变为下拉多选。
3. 下拉选项来自内置 `币种映射表.xlsx` 的 C 列。
4. 至少勾选 1 个币种才能生成。

### 生成规则

1. 会生成开户日期当日的一条记录。
2. 还会继续生成 `开户日期到今天之间所有月末日期` 的记录。
3. 如果开户日期本身是月末，也会保留开户日这一条。
4. 所有生成行中：
   1. `银行名称`、`所在地`、`银行账号` 取输入框内容。
   2. `币种` 取单币种输入值，或多币种勾选值。
   3. `账单日期` 取开户日或对应月末日期。
   4. `期末余额` 固定为 `0`。
   5. `期初余额`、`期初可用余额`、`期末可用余额` 默认留空。

### 导出命名规则

`银行名称-所在地-银行账号-币种-NEW_BALANCE.xlsx`

如果是多币种账户，则文件名里的 `币种` 固定输出为 `多币种`。

## 五、状态框与报错文件

### 1. 状态框

主模块和新开账户模块都有各自独立的状态框。

### 2. 状态框点击行为

#### 主模块

1. 如果当前是报错状态：
   1. 点击状态框会导出报错文件。
2. 如果当前是“补录上一账单日余额”提示：
   1. 点击状态框会打开余额补录窗口。

#### 新开账户模块

1. 如果该模块出现报错：
   1. 点击状态框会导出报错文件。

### 3. 报错文件

如果出现校验错误或系统异常，软件会自动生成详细报错文件。

报错文件命名规则：

`YYYYMMDD-HHMMSS-模板名-错误步骤.txt`

文件中会尽量写清楚：

1. 是哪一步出错
2. 为什么出错
3. 哪一行、哪个字段有问题
4. 相关上下文信息

## 六、背景设置

1. 点击左下角 `🎨` 按钮可打开背景设置面板。
2. 可直接在全色谱上点选颜色。
3. 可点击 `导入背景文件` 替换应用背景图。
4. 可点击 `完成` 保存背景。
5. 可点击 `重置` 恢复默认背景，重置前会有确认提醒。

### 背景图片限制

1. 支持格式：`PNG / JPG / JPEG / WEBP`
2. 文件大小：不超过 `5MB`
3. 最小分辨率：`1200 x 700`
4. 最大分辨率：`4096 x 4096`

## 七、文件与数据位置

软件运行过程中会使用以下目录：

1. 导出文件目录：  
   `文档/网银账单生成小助手/exports/日期/`
2. 日志目录：  
   `文档/网银账单生成小助手/logs/`
3. 报错文件目录：  
   `文档/网银账单生成小助手/error-reports/日期/`
4. 本地余额种子目录：  
   `文档/网银账单生成小助手/balance-seeds/`
5. 运行日志文件：  
   `文档/网银账单生成小助手/app_activity_log.txt`

## 八、常见注意事项

1. 软件已内置 `COMMON枚举.xlsx`，不需要首次手动导入枚举表。
2. 如果重复导入同名模板，旧映射关系会被重置，需要重新检查映射。
3. 模板名称建议采用 `银行名称-所在地` 的格式，否则 `Channel` 和余额账单中的 `银行名称 / 所在地` 可能无法按预期拆分。
4. 如果 `Currency` 原值不是英文，建议优先确认是否能在内置币种映射表中匹配到。
5. 如果导入后状态框提示可补录余额，说明明细账单通常已经生成，只是余额账单还缺少上一账单日余额。
6. 如果状态框提示报错但 `导出明细` 仍可点击，说明明细文件已生成，只是存在需要人工确认的异常。

## 九、建议的标准操作顺序

### 第一次使用网银账单生成模块

1. 导入模板
2. 打开 `模板管理`
3. 维护 `映射关系管理`
4. 如有需要，维护 `账户映射`
5. 选择模板
6. 导入原始账单文件
7. 查看状态框提示
8. 导出明细 / 导出余额

### 处理“首次导入余额”场景

1. 导入账单文件
2. 若状态框提示需要补录上一账单日余额，则点击状态框
3. 填写上一账单日日期和余额
4. 点击 `完成`
5. 等待系统自动重试余额校验
6. 校验成功后再导出余额账单

### 处理新开但无交易账户

1. 切换到 `新开账户生成网银账单`
2. 填写必填项
3. 如是多币种账户，先勾选并选择币种
4. 点击 `生成`
5. 点击 `导出`

## 十、如果出现问题

建议按以下顺序检查：

1. 模板是否选对
2. 映射关系是否维护完整
3. `BillDate` 是否已映射
4. 如果要生成余额账单，`Balance` 是否已启用
5. 如果要生成余额账单，`MerchantId` 是否已映射且导入值不为空
6. `Currency` 原值是否能匹配币种映射表
7. 状态框当前是“补录余额”还是“导出报错文件”

如需进一步排查，建议优先提供：

1. 原始账单样例
2. 使用的模板文件
3. 导出的报错文件
