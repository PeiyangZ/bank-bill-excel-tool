# Version Rollback

## 原则

- 每次版本迭代必须同步更新 `CHANGELOG.md`。
- 每次发布安装包时应保留对应版本号的构建产物。
- 回溯时优先通过安装包版本回退，避免直接操作用户数据库文件。

## 建议流程

1. 为当前提交打版本标签，例如 `v1.0.0`。
2. 通过 `npm run dist:win` 产出带版本号的安装包并归档。
3. 若需回退，切换到目标标签重新构建，或直接分发归档安装包。
4. 如涉及数据结构变化，先备份 Electron `userData` 目录内的 `tool-data.sqlite`。

## 数据回退建议

- 当前版本未引入破坏性数据库迁移。
- 后续如引入字段或表变更，应在发布说明中增加数据库变更说明和回退步骤。
